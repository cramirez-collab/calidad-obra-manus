## Niveles de Severidad

| Nivel | Descripción | Acción Requerida |
|-------|-------------|------------------|
| **Leve** | Defecto estético menor | Corregir antes de entrega |
| **Moderado** | Afecta funcionalidad parcialmente | Corregir en 48 horas |
| **Grave** | Afecta funcionalidad significativamente | Corregir en 24 horas |
| **Crítico** | Riesgo de seguridad o estructural | Corregir inmediatamente |

## Notas de Implementación

1. **Especialidades**: Se recomienda crear las especialidades base al iniciar un proyecto y agregar las específicas según el tipo de obra.

2. **Atributos**: Los atributos deben asociarse a las especialidades correspondientes para facilitar las inspecciones.

3. **Defectos**: El catálogo de defectos permite generar estadísticas consistentes. Se recomienda:
   - Usar los códigos estándar para facilitar reportes
   - Asociar cada defecto a su especialidad
   - Mantener los tiempos estimados actualizados según experiencia

4. **Personalización**: Este catálogo es una base que puede personalizarse según:
   - Tipo de proyecto (residencial, comercial, industrial)
   - Normativas locales
   - Requerimientos del cliente
